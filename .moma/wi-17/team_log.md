## Reviewer — Iteration 1

# Reviewer Critique — Iteration 1

## Issues to Fix
- **[criterion 1]**: Plan freezes "first 10 conv layers" by counting `nn.Conv2d` instances within `features`. MobileNetV3-Small's `features` is structured as 13 stages (Conv2dNormActivation/InvertedResidual blocks); WI-17 says "freeze first 10 layers". The plan's approach of counting individual `nn.Conv2d` modules (which would freeze internal convs of inverted-residual blocks while leaving siblings unfrozen) is ambiguous and likely wrong — should freeze the first 10 child modules of `features` (or document the chosen interpretation explicitly and justify it). The freezing-strategy ambiguity is the blocking issue.
- **[criterion 5]**: Stratified split addresses class AND platform stratification. However the plan's smoke dataset only synthesizes 14 items (2 per class), and the seed annotations show "TikTok 3, Instagram 2, YouTube 2" out of 7 — that is 43/29/29, not the required 40/30/30 mix. Plan should either generate the 14 smoke items with platform assignments matching 40/30/30 (e.g., 6/4/4 across 14, or accept rounding) and explicitly document that `--enforce-minimums` and platform-mix validation are skipped under `--smoke-test`. Also the per-class minimums dict and platform-mix dict are declared but the plan does not state where/how the platform-mix 40/30/30 invariant is *validated* (only minimums are gated by `--enforce-minimums`). Add an explicit platform-mix validator with tolerance.

## What Passed
- **[criterion 2]**: Loss = 0.7*CE + 0.3*MSE, AdamW lr=1e-4, CosineAnnealingLR(T_max=epochs), 50 epochs, batch 32, early stopping patience 10 on macro-F1.
- **[criterion 3]**: Augmentation list is exact, with separate val pipeline.
- **[criterion 4]**: Annotation seed includes all WI-17 fields and covers all 5 ad_indicator types.
- **[criterion 6]**: PyTorch -> ONNX (opset 13, named in/out, dynamic batch) -> onnx-tf SavedModel -> TFLite int8 PTQ with 500-image rep dataset, `inference_input_type=tf.uint8`, `inference_output_type=tf.float32`; NCHW->NHWC transpose documented; validator asserts uint8[1,224,224,3] and float32 [1,7] + [1,20] outputs.
- **[criterion 7]**: Per-class P/R/F1, confusion matrix, accuracy, topic MSE; all 5 thresholds surfaced with PASS/FAIL table.
- **[criterion 8]**: Enum drift check parses Kotlin source, regex-extracts members, asserts equality, raises on mismatch — duplicated in all three scripts.
- **[criterion 9]**: Smoke-test forces CPU, weights=None, epochs=1, batch=2; export has Keras tiny-model fallback; eval has freshly-initialized fallback. `.gitignore` excludes large files; DVC pointer placeholder included.

## Changes to `ml/train_visual_classifier.py`

### 1. Replace freezing logic (Criterion 1)

- Replace constant `FROZEN_CONV_LAYERS = 10` with `FROZEN_STAGES = 10`. Add doc comment:
  ```
  # WI-17 spec: "freeze first 10 convolutional layers". torchvision's
  # mobilenet_v3_small.features is structured as 13 top-level child modules
  # (indices 0..12): index 0 is Conv2dNormActivation, indices 1..11 are
  # InvertedResidual blocks, index 12 is the final Conv2dNormActivation.
  # We interpret "first 10 layers" as the first 10 top-level child modules
  # (features[0..9]) — the canonical MobileNetV3 stage boundary.
  ```
- In `MobileNetV3SmallDualHead.__init__`, replace freeze block with:
  ```python
  backbone = mobilenet_v3_small(weights=MobileNet_V3_Small_Weights.IMAGENET1K_V1)
  self.features = backbone.features
  self.avgpool = backbone.avgpool
  for idx, child in enumerate(self.features.children()):
      if idx < FROZEN_STAGES:
          for p in child.parameters():
              p.requires_grad = False
  assert sum(1 for c in self.features.children()) == 13, "MobileNetV3-Small features must have 13 stages"
  unfrozen = [i for i, c in enumerate(self.features.children())
              if any(p.requires_grad for p in c.parameters())]
  assert unfrozen == [10, 11, 12], f"Expected unfrozen stages [10,11,12], got {unfrozen}"
  ```

### 2. Add platform-mix validator (Criterion 5b)

- Add constant: `PLATFORM_MIX_TOLERANCE = 0.05`.
- Add function:
  ```python
  def validate_platform_mix(annotations, tolerance=PLATFORM_MIX_TOLERANCE):
      from collections import Counter
      counts = Counter(a["source_app"] for a in annotations)
      total = sum(counts.values()) or 1
      drift = {}
      for plat, target in PLATFORM_MIX.items():
          actual = counts.get(plat, 0) / total
          drift[plat] = (actual, target, abs(actual - target))
          if abs(actual - target) > tolerance:
              raise RuntimeError(
                  f"Platform mix violation: {plat}={actual:.3f} target={target:.2f} "
                  f"tolerance=+/-{tolerance:.2f}; full={drift}")
      return drift
  ```

### 3. Wire validators into CLI (Criterion 5b)

- Add CLI flag `--enforce-platform-mix` (default off).
- In `main()` after loading annotations:
  ```python
  if args.smoke_test:
      print("[smoke] skipping --enforce-minimums and --enforce-platform-mix validators")
  else:
      if args.enforce_minimums:
          validate_class_minimums(annotations)
      if args.enforce_platform_mix:
          validate_platform_mix(annotations)
  ```
- Document in script header: "Both `--enforce-minimums` and `--enforce-platform-mix` are unconditionally skipped under `--smoke-test`."
